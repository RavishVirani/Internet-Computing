8. List

We frequently make and use lists to make a context clear. There are two kind of lists, unordered list, and ordered list. A unordered list looks like this.

A computer has five components:

  • CPU
  • Storage device such as memory and hard drive
  • Output device such as a display
  • Input device such as a keyboard
  • Programs such as an operating system

An ordered list looks like the following. It contains a nested ordered and unordered lists.

A computer has five components:

  1. CPU
  2. Storage device
    1. memory
    2. hard drive
  3. Output device such as a display
  4. Input device
    • keyboard
    • mouse
  5. Programs such as an operating system

More general list can be made by definition.

152
Two-place trainer
172
Smaller four-place airplane
182
Larger four-place airplane
210
Six-place airplane - high performance

10. Form

Forms are an important entities in client programming. We cab use form to submit our request for the execution of a a server program.

1. Text

Name:

Password
 

This field does not provide any true security; it simply provides basic protection against someone looking over the shoulder of the user. The value is not encrypted when it is transferred to the web server, which means that passwords are displayed as part of the query string for GET requests.

2. Hidden field

Hidden fields are not visible to the user. They are generally used only with forms which are themselves generated by a CGI script and are useful for passing information between a series of forms. <INPUT TYPE="hidden" NAME="username" VALUE="msmith">  

Like password fields, hidden fields provide no security. Users can view the name and value of hidden fields by viewing the HTML source in their browsers.
